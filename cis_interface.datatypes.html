class cis_interface.datatypes.CisBaseType.CisBaseType(**typedef)[source]¶

Bases: object

Base type that should be subclassed by user defined types. Attributes should be overwritten to match the type.

Parameters:**kwargs – All keyword arguments are assumed to be type definition properties which will be used to validate serialized/deserialized messages.
name¶

str – Name of the type for use in YAML files & form options.

description¶

str – A short description of the type.

properties¶

dict – JSON schema definitions for properties of the type.

definition_properties¶

list – Type properties that are required for YAML or form entries specifying the type. These will also be used to validate type definitions.

metadata_properties¶

list – Type properties that are required for deserializing instances of the type that have been serialized.

data_schema¶

dict – JSON schema for validating a JSON friendly representation of the type.
